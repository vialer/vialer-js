    async _generateVaultKey(username, password) → {Promise}

    The session key is generated from the user's password and is cached in-memory. The user has to refill it's password as soon the plugin is restarted. All application data is decrypted using the session key.

    Parameters:

    async loadIdentity(username, password, e2e)

    An identity is foremost a sessionkey that is used to encrypt locally stored data with. It can also provide means to setup a secure communication channel with between two endpoints. In that case a transient ECDH key and a static RSA-PSS key is used to negotiate a PFS secret key between the two endpoints.

    Parameters:
    Name Type Default Description
    username String

    The username to unlock local data with.
    password String

    The password to unlock local data with.
    e2e Boolean false

    Whether to create an asymmetric encryption key.

    async storeVault()

    Persist the encryption key to the unencrypted part of the store, so the plugin can automatically unlock. This is a tradeoff between usability and security.
